# 一个视频搞懂 OpenClaw — 飞天闪客深度解析

*基于B站UP主「飞天闪客」视频《一个视频搞懂OpenClaw！》整理*

---

## 核心观点

> **OpenClaw 的成功不是因为它发明了什么新技术，而是它在最合适的时间点，把现有的东西用最合适的形式组合了起来——造了一座连接 AI Agent 和普通人的桥。**

---

## 第一层：起源——一个意外

奥地利程序员 Peter 原本只想把 Claude Code 接到聊天软件里，这样出门在外也能操控家里的 AI。

结果有一次他随手发了条语音——**他根本没给 AI 加语音理解功能**——AI 却自己给自己写了个语音转文字模块，把语音听懂了还回复了一条消息。

这就是"智能体"的神奇效果：**它不知道自己会什么，但它能学。**

---

## 第二层：它真正做了什么——造桥

Peter 并不是新技术的发明者，他的贡献是——**造桥**。

| 之前 | OpenClaw 之后 |
|------|--------------|
| AI Agent 只在程序员圈子里流行 | 把 AI 接入了 20+ 聊天平台（微信、飞书、Discord...） |
| 需要用命令行操作 | 像发微信一样操作 AI |
| 普通人觉得 AI 就是聊天机器人 | 普通人第一次觉得 AI 真的在"帮我干活" |

---

## 第三层：实战效果

UP主实测了几个任务：

### 删除 Chrome 浏览历史
> AI 不是去浏览器里一个个点，而是**直接找到 Chrome 存历史记录的本地 SQLite 文件**，几行指令下去连缓存都清干净了。120分完成。

### 整理桌面文件
> 乱七八糟的文件夹，几分钟分门别类整理好。

### 下载网课视频
> OpenClaw 先是被网站拒绝 → 试了 ytdlp 下错了格式 → 发现自己下错了 → 又发现 UP主之前下过这套课 → 跑过来问"到底谁给谁干活" → 最后还是自己把正确链接搞到了。

**核心循环**：先调用工具 → 再检查结果 → 对了就继续，错了就重来。

---

## 第四层：OpenClaw 的技术架构

| 组件 | 作用 |
|------|------|
| **基础模型** | 权衡了性能和成本，选了 MiniMax M2.5（而不是最强的 Claude Opus 4.6） |
| **IX 核心工具** | 直接执行各种命令行命令（Shell Code 精神的延续） |
| **RED** | 读取各种文件 |
| **Process** | 获取任务进度 |
| **Sessions History** | 调取对话历史 |
| **Skills 生态** | 一份份提前写好的"提示词说明书"，教 AI 具体怎么做事 |

---

## 第五层：Skills 生态才是护城河

> **Skills 本质上就是提前写好的提示词**，告诉 AI 在什么情况下用什么工具、怎么用。

- 全世界上万人一起贡献 Skills
- GitHub 上有个 Awesome OpenClawSkills 项目，整理了 **2800+ 个 Skills**
- 还有专门的平台 ClawHub，**将近 6000 个 Skills**
- 甚至 AI **自己**都能创建 Skills——清浏览记录那次，OpenClaw 就是自己给自己写了份 Skills 说明书

> UP主引用了《黑客帝国》的梗：往脑子里传数据就能学会格斗——今天在 AI Agent 身上已经实现了。

---

## ⚠️ 安全问题：被严重低估

UP主把 OpenClaw 接入飞书让同事体验，结果一顿"聊天追问"就把 **API 密码、网关令牌**等敏感信息全被套出来了。换上更强的 Claude Opus 4.6，OpenClaw 照样"老实交代"。

| 威胁 | 数据 |
|------|------|
| 已检测到的恶意 Skills | **1184 个** |
| 存在安全漏洞的 Skills | **36.8%** |

如果有人在 Skills 里埋恶意代码，AI 会在你不知情的情况下自动发送隐私信息和代码——**每台配置了 OpenClaw 的电脑都可能是定时炸弹。**

---

## ⚠️ 上下文灾难：Agent 的能力天花板

UP主布置了一个复杂任务：
> 从网上找50张不同做法的小龙虾图片 → 存到桌面文件夹 → 做成表格发给我确认 → 我选中后打包发到飞书群

OpenClaw 干了整整 **8个多小时**，最后：
- 只发来了文件路径而不是真正的图片
- 文件夹里只有一张是小龙虾，其他全是别的菜
- **它在中途就已经忘了自己到底要找什么**

这就是"**上下文灾难**"——每次对话要给 AI 灌输大量信息（人设、工具列表、用户画像、记忆等），你的指令只占 5% 左右，AI 在这么多信息里**漏掉你的需求、选错工具**是家常便饭。

> Meta 高管被 OpenClaw 删光邮件，也是因为任务太长，AI 把最开始的指令忘了。

|------|------|

## 终极总结

**Peter 本人说：**
> OpenClaw 不是魔法。把它拆开，每个步骤都不过如此。但有时候把现有的东西排列组合，再加上一点新点子，神奇的事情就发生了。

**UP主补充：**
> OpenClaw 的成功在于：AI 浪潮到了这个节点，所有条件都为它准备好了。它展示了很多人心中最有科幻感的场景——**拿着手机发一条消息，千里之外就有一个 AI 帮你全自动做好任何事情。**

**但现实是：**
- 它现在只适合简单重复的日常杂活
- 复杂任务它还是个"走一步算一步的新兵蛋子"
- 安全问题被严重低估，36.8% 的 Skills 有漏洞

---

*欢迎来到智能体的时代——但路还长。*
