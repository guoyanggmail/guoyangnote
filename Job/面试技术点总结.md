# Android面试技术点总结

## 一、Android基础
### 1. View相关
#### 1.1 View的绘制流程
- **问题**：View的绘制流程是什么？为什么不能在子线程进行View的绘制？
- **答案**：
  1. View的绘制流程分为三个步骤：
     - measure: 测量视图大小
     - layout: 确定视图位置
     - draw: 绘制视图
  2. 不能在子线程绘制View的原因：
     - Android的UI线程是单线程模型
     - ViewRootImpl的checkThread()会检查是否在主线程
     - 避免并发导致的线程安全问题

#### 1.2 事件分发机制
- **问题**：事件分发机制和事件拦截机制？
- **答案**：
  1. 事件分发流程：Activity -> Window -> DecorView -> ViewGroup -> View
  2. 三个重要方法：
     - dispatchTouchEvent(): 分发事件
     - onInterceptTouchEvent(): 拦截事件(ViewGroup特有)
     - onTouchEvent(): 处理事件
  3. 事件拦截规则：
     - 如果ViewGroup拦截DOWN事件，后续事件不会再传递给子View
     - 子View可以通过requestDisallowInterceptTouchEvent()阻止父View拦截事件

#### 2.1 启动优化
- **问题**：App启动优化如何做？

#### 2.2 内存优化
- **问题**：内存泄露的原因及解决方案？如何进行内存优化？
- **答案**：
  1. 内存泄露原因及解决：
     - 静态变量持有Activity引用：
       ```kotlin
       // 错误示例
       class MainActivity : AppCompatActivity() {
           companion object {
               private var context: Context? = null  // 静态变量
           }
           override fun onCreate(savedInstanceState: Bundle?) {
               super.onCreate(savedInstanceState)
               context = this  // 持有Activity引用
           }
       }
       
       // 正确做法
       class MainActivity : AppCompatActivity() {
           companion object {
               private var contextRef: WeakReference<Context>? = null  // 使用弱引用
           }
           override fun onCreate(savedInstanceState: Bundle?) {
               super.onCreate(savedInstanceState)
               contextRef = WeakReference(applicationContext)  // 使用Application Context
           }
       }
       ```

     - Handler内存泄露：
       ```kotlin
       // 错误示例
       private val handler = Handler()
       
       // 正确做法
       class MyHandler : Handler() {
           private val activityRef: WeakReference<Activity>
           
           override fun handleMessage(msg: Message) {
               val activity = activityRef.get()
               if (activity != null) {
                   // 处理消息
               }
           }
       }
       ```

     - 匿名内部类/线程：
       ```kotlin
       // 错误示例
       thread {
           // 长时间运行的操作
       }
       
       // 正确做法
       private val job = Job()
       private val scope = CoroutineScope(job + Dispatchers.Main)
       
       override fun onDestroy() {
           super.onDestroy()
           job.cancel() // 取消协程
       }
       ```

  2. 内存优化工具：
     - LeakCanary使用：

#### 2.1 OkHttp
- **问题**：OkHttp的核心原理？如何实现拦截器链？
- **答案**：
  1. 责任链模式实现：
     ```kotlin
     // 1. 拦截器接口
     interface Interceptor {
         fun intercept(chain: Chain): Response
     }
     
     // 2. 拦截器链
     class RealInterceptorChain(
         private val interceptors: List<Interceptor>,
         private val index: Int,
         private val request: Request,
         private val call: Call
     ) : Chain {
         override fun proceed(request: Request): Response {
             // 调用下一个拦截器
             val next = RealInterceptorChain(interceptors, index + 1, request, call)
             val interceptor = interceptors[index]
             return interceptor.intercept(next)
         }
     }
     ```

  2. 常用拦截器实现：
     ```kotlin
     // 1. 重试拦截器
     class RetryInterceptor : Interceptor {
         override fun intercept(chain: Chain): Response {
             val request = chain.request()
             var response: Response? = null
             var retryCount = 0
             
             while (response == null && retryCount < MAX_RETRY) {
                 try {
                     response = chain.proceed(request)
                 } catch (e: Exception) {
                     retryCount++
                 }
             }
             return response ?: throw IOException("Request failed")
         }
     }
     
     // 2. 缓存拦截器
     class CacheInterceptor(private val cache: Cache) : Interceptor {
         override fun intercept(chain: Chain): Response {
             // 读取缓存
             val cacheResponse = cache.get(chain.request())
             if (cacheResponse != null && !cacheResponse.isExpired()) {
                 return cacheResponse
             }
             
             // 网络请求
             val response = chain.proceed(chain.request())
             // 保存缓存
             cache.put(chain.request(), response)
             return response
         }
     }
     ```
